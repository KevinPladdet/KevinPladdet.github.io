What's the game about?

Dungeon Explorer is a 3D game about exploring a dungeon to uncover treasures and to defeat monsters.

The game has polygon assets for all of the game objects and has controller support as well.

Why did I make this project?

Making a 3D game takes a lot of work and usually I don't manage to use assets for everything. The goal of this game was to make a nice exploration / adventure game that has assets for every game object.

I also wanted to try and add controller support, because it's pretty cool and will be usesful to learn for future projects.

What did I learn?

I learned how to get assets and how to use them

The Unity Asset Store is perfect for finding asset packs, because there are thousands on there.
I also used Humble Bundle to buy a polygon assets bundle, which I ended up using for the game.

Scriptable Objects

Scriptable objects are really nice to use, because it keeps the code clean. I used scriptable objects to give each enemy different stats, like health, movement speed, etc.

Main Menu / Settings Menu

In most, if not all of my games I have a main menu and settings menu. They're all pretty basic, but they function. Isn't that what matters?

Wrong. For Dungeon Explorer I acted like the main menu and settings menu was the most important thing ever.

My goals were the following:

- Moving background on the main menu & settings menu
- Be able to access the settings menu from the game scene
- FOV slider + sensitivity slider

I had already made a FOV slider and sensitivity slider for Redo, so I did the same things to make it work for Dungeon Explorer.

In every game I made up until now, my main menu and settings menu were in a different scene than the game scene. This was a problem, because if you would go back to the main menu, it would not save your progress on the game scene.

To fix this I made the main menu and settings menu in the same scene as the game. This also helped me make the moving backgrounds, because I added 2 cameras that show the beautiful atmosphere. Depending on what menu you're in would decide what camera enables.

Controller Support

To add controller support I used Unity's new input system. With this I could make multiple control profiles, one for mouse and keyboard and the other for controller.

I had a bit of an annoying time trying to figure out how it works, but once it clicked I didn't mind it too much.

Even when you interact with objects in the game, if you have a controller connected it will display the controller interaction button.
