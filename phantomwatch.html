What's the game about?

Inspired by “I'm on Observation Duty”, which is a game about detecting & banishing anomalies.

You are a Phantom Watch employee. Your job is to carefully inspect the facility through the security system.
Any paranormal activity must be identified and banished.

Can you make it through the night?

Team Exam

I made this game for a school exam together with 3 friends. For the exam we had to create a game with design documents and also document others playtesting it.

We started with creating a FDD (functional design document) which contains everything about the game.

After the FDD, we created a TDD (technical design document) where we explain how we're going to code everything.

During the exam we let others playtest the game, which gave us valuable feedback on what we could improve on.

We had 3 sprints in total (1 week per sprint) and used Notion to plan everything.

What did I learn?

Notion

I have never used Notion before, but one of my friends had some experience with it and said it was good.

On Notion you can create documents very easily, so we had the FDD and TDD + the kanban board all together.
It also includes live updates on changes, so it was easy to work together in Notion.

The planning went well too, since we first made all the tasks and then divided them. During the project there were some sick days between others and one of their laptops broke down too, but we still managed to create effective solutions for that.

Handling merge conflicts

To prevent merge conflicts, everyone had their own folders for scripts/assets/scenes/etc. This made merge conflicts impossible, unless we started working in eachothers folders.

We would always ask eachother if it was safe to work in them, so we never got a merge conflict during the entire project.

Other Information

Phantom Watch took us 3 weeks to make in Unity (around 15 hours per week) and you can play it on Windows.
